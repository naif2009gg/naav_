# موقع تقديم إدارة naav + بوت ديسكورد (قبول / رفض)

## الفكرة بالكامل
- الموقع: نموذج تقديم بخطوات (Wizard) مبني بـ HTML/CSS/JS عادي.
- الأسئلة + الإجابات الصحيحة محفوظة في ملف على السيرفر فقط:
  `netlify/functions/_questions-data.js` — **لا يصل هذا الملف أبدًا لمتصفح المتقدم**،
  المتصفح يستقبل فقط نص السؤال والخيارات بدون أي إشارة للإجابة الصحيحة (عبر `/api/questions`).
- عند الإرسال (`/api/submit`) يتم تصحيح الأسئلة ذات الخيارات آليًا، ثم تُرسل النتيجة كـ Embed
  داخل روم محدد بالديسكورد مع زرّين: **قبول** و **رفض**.
- لا حاجة لتشغيل بوت دائم (24/7) على سيرفر خاص. كل شيء يعمل كـ Netlify Functions
  (سيرفرليس) — مناسب 100% لاستضافة Netlify.
- عند ضغط أي إداري على "قبول" أو "رفض" يقوم Discord بإرسال الطلب مباشرة لدالة
  `discord-interactions.js` (هذا ما يسمى Interactions Endpoint URL)، والتي:
  1. تتحقق من توقيع الطلب (يمنع أي شخص من تزييف الطلبات).
  2. تعطي/تسحب الرتبة المناسبة للمتقدم.
  3. ترسل له رسالة خاصة عند القبول فقط.
  4. تعدّل نفس الرسالة في الروم وتحذف الأزرار وتسجّل اسم من اتخذ القرار.

---

## 1) إنشاء البوت في Discord
1. اذهب إلى https://discord.com/developers/applications وأنشئ تطبيقًا جديدًا.
2. من تبويب **Bot**: فعّل البوت، وانسخ **Token** (هذا هو `DISCORD_BOT_TOKEN`).
   - فعّل صلاحية **Server Members Intent** من نفس الصفحة.
3. من تبويب **General Information**: انسخ **Public Key** (هذا هو `DISCORD_PUBLIC_KEY`).
4. من تبويب **OAuth2 → URL Generator**: اختر صلاحيات `bot` مع:
   - Manage Roles
   - Send Messages
   - Embed Links
   - استخدم الرابط الناتج لدعوة البوت لسيرفرك.
5. **مهم جدًا:** رتبة البوت في إعدادات السيرفر (Roles) يجب أن تكون **أعلى** من رتبتي
   القبول والرفض، وإلا لن يستطيع منحهما.

## 2) جلب المعرّفات (IDs) المطلوبة
فعّل Developer Mode من إعدادات ديسكورد (Settings → Advanced)، ثم اضغط يمين واختر "Copy ID" على:
- **السيرفر (Guild)** → `DISCORD_GUILD_ID`
- **روم الإدارة** الذي تريد إرسال الطلبات فيه → `DISCORD_CHANNEL_ID`
- **رتبة القبول** → `ACCEPT_ROLE_ID`
- **رتبة الرفض** → `REJECT_ROLE_ID`

## 3) رفع المشروع على Netlify
1. ارفع هذا المجلد كمشروع على GitHub (أو اسحبه مباشرة في Netlify Drag & Drop —
   لكن يفضّل GitHub لإدارة التحديثات).
2. في Netlify: **Add new site → Import an existing project**، واختر المستودع.
3. إعدادات البناء (موجودة مسبقًا في `netlify.toml`):
   - Publish directory: `public`
   - Functions directory: `netlify/functions`
4. من **Site settings → Environment variables** أضف:

   | المتغير | القيمة |
   |---|---|
   | `DISCORD_BOT_TOKEN` | توكن البوت |
   | `DISCORD_PUBLIC_KEY` | المفتاح العام للتطبيق |
   | `DISCORD_GUILD_ID` | آيدي السيرفر |
   | `DISCORD_CHANNEL_ID` | آيدي روم الطلبات |
   | `ACCEPT_ROLE_ID` | آيدي رتبة القبول |
   | `REJECT_ROLE_ID` | آيدي رتبة الرفض |
   | `PING_ROLE_ID` | آيدي الرتبة التي تريد إشعارها مع كل طلب جديد (مثل رتبة الإدارة) — اختياري |

5. اعمل **Deploy site**. بعد الانتهاء سيعطيك Netlify رابطًا مثل:
   `https://naav-apply.netlify.app`

## 4) ربط رابط التفاعلات (Interactions Endpoint URL)
1. رجع لصفحة تطبيق البوت في Discord Developer Portal → **General Information**.
2. في خانة **Interactions Endpoint URL** ضع:
   ```
   https://YOUR-SITE.netlify.app/.netlify/functions/discord-interactions
   ```
3. اضغط **Save Changes** — إذا ظهرت علامة ✅ يعني التحقق نجح والربط شغّال.
   (إذا فشل، تأكد إن متغيرات البيئة محفوظة والـ deploy تم بنجاح قبل المحاولة.)

## 5) جرّب الموقع
- افتح رابط الموقع → اضغط "ابدأ التقديم" → جاوب على الأسئلة → أرسل.
- يفترض أن تظهر رسالة Embed في الروم المحدد مع زرّي قبول/رفض.
- جرّب الضغط على أحدهما، وتابع:
  - تغيّر الرتبة للمتقدم في السيرفر.
  - رسالة خاصة له عند القبول.
  - تعديل الرسالة الأصلية وحذف الأزرار.

## تعديل الأسئلة
الأسئلة موجودة في ملف واحد:
```
netlify/functions/_questions-data.js
```
كل الأسئلة حاليًا مفتوحة (نصية) — يجاوب عليها المتقدم بنفسه والإدارة تقيّمها يدويًا.
أضف/عدّل/احذف بحرية بنفس الشكل: `{ id: "qX", label: "نص يظهر في ديسكورد", question: "نص يظهر بالموقع" }`.

**ملاحظة:** بالإضافة لأسئلة الاستبيان، الموقع يطلب من المتقدم **آيدي حسابه بالديسكورد**
كحقل أول إلزامي (غير ظاهر داخل ملف الأسئلة، مكتوب مباشرة بالواجهة) — هذا الحقل ضروري
ليقدر البوت يعطيه الرتبة ويرسل له رسالة القبول/الرفض الخاصة.

## شكل الرسالة في ديسكورد
الرسالة تُرسل بهذا التنسيق تمامًا كما طلبت:
```
||@everyone||
||<@&PING_ROLE_ID>||

📋 استبيان المقابلات - طلب جديد
**الاسم :**
...
**العمر :**
...
**1- خبرتك في فايف إم :**
...
... (باقي الأسئلة)
**النتيجة :** قيد المراجعة   ← تتحول تلقائيًا إلى ✅ مقبول أو ❌ مرفوض بعد ضغط أي إداري على الزر
**عدد الأسئلة المجاوب عليها :** 9 / 9
المتقدم: @اسم_المتقدم
```

## ملاحظة عن "التشفير"
- نص الأسئلة نفسه يصل للمتصفح لأن المتقدم لازم يقرأها ليجاوب عليها — هذا أمر طبيعي
  ولا يوجد طريقة لإخفاء نص سؤال عن الشخص المطلوب منه الإجابة عليه. لكن لا يوجد أي
  "إجابة صحيحة" مخزّنة بالكود يقدر أحد يسرقها، لأن كل الأسئلة مفتوحة وتُقيَّم يدويًا.
- تمت إضافة رؤوس حماية أساسية (Security Headers) في `netlify.toml`.
